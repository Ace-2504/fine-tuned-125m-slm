01The finding: quality plateaus, forgetting doesn't

04Why it plateaus

Perplexity is not quality. ~40% of training is closed-book QA — recall a document fact from a single exposure, which a 125M model largely can't do. Common facts slowly land; specific ones don't.

Q · What is the standard of proof in a civil lawsuit?
Day 1–2: circular repetition → Day 3: "…is the preponderance of the evidence standard." ✓ a common fact finally learned
Q · Summarize what a 10-K annual report contains.
Day 3: "Summarize the annual report for the year ended December 31, 1999." (still echoes / no real summary)
Q · What does it mean for a contract clause to be severable?
Day 3: "The contract clause must be severable, and the parties must be bound by the terms…" (muddled)

The judge set is drawn mostly from closed-book recall — the one thing this model can't learn — so it measures the model's worst skill while the learnable grounded tasks (summarize/extract /rewrite/RAFT) go under-measured. See the training feedback for the fix.

05What next — the v2 pivot

The study answered its question: on this mix, scaling data plateaus quality. The evidence now points to a design change rather than more days.

  • Rebalance to grounded tasks — context-in-prompt QA + summarize/extract/rewrite, which a 125M model can learn — and restrict closed-book QA to general knowledge. (Restarts the curve as v2.)
  • Report the judge per mode (grounded vs recall) so scoring reflects what the model can actually do.
  • Fix decoding — repetition penalty + no-repeat-ngram (removes greedy loops; comparability-safe).
  • Bound forgetting if continuing the current mix — mix in a little raw pretraining data (replay).

06Method

  • Base: LlamaForCausalLM, 12L / 768d / 12h, 125.8M params, vocab 16,384, context 1,024, tied embeddings — pretrained from scratch on 2.04B tokens (val ppl 11.35).
  • Teacher: gemini-3.1-flash-lite generates grounded QnA whose answers are stated only in the source passage; free tier, $0.
  • Training: full fine-tune, 3 epochs, batch 16, LR 3e-5→3e-6 cosine, AdamW, bf16, assistant-only loss masking, seed 1337.
  • Design: each day re-initialises from the base checkpoint on the cumulative dataset — the only variable is dataset size, so the curve isolates the effect of data.
  • Eval: SFT-eval perplexity (task fit) · retention perplexity on pretraining val (forgetting) · Gemini-as-judge 1–5 (answer quality) · fixed greedy sample generations.
Day-3 run: 3,000 pairs · 564 steps · 241 s · Modal L4 → ~$0.064. Cumulative GPU spend across days 0–3: ~$0.17 (teacher $0, free tier).
